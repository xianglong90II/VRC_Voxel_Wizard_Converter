# 基于2D纹理的体素渲染方案 - 正式报告与资料
## 项目背景与目标

本方案旨在实现单色体素物体的高效渲染，采用2D纹理切片技术替代传统的3D纹理存储方式。通过利用2D纹理的多通道特性，实现体素空间的有效表达，降低显存占用，提高渲染性能。

## 技术原理

- 采用单色体素表示，利用纹理的单通道黑白值表示体素开关状态。

- 利用RGBA四个通道同时存储四层体素信息，提升数据密度。

- 通过切片技术，将大尺寸2D纹理划分为多个小切片，每个切片对应一定数量的体素层。

## 体素空间计算

- 纹理尺寸：宽度 W 乘以 高度 H，例如 256×256。

- 切片尺寸：边长 S 乘以 边长 S。

- 通道数：C=4（RGBA）。

- 切片数量：N = (W ÷ S) × (H ÷ S)。

- 总层数（深度）：Z = N × C。

## 位运算优化

传统方案每通道存储 0 或 1，单像素代表 4 层体素。

位运算方案利用每通道 8 位存储，单像素可代表 32 层体素。

极限情况下，256×256 纹理可表现 128×128×128 体素空间。

## 数学推导

设体素正方体边长为 L，满足：

[ \frac{W^{2}}{L^{2}} \times C = L ]

[ L = \sqrt[3]{W^{2} \times C} ]

以 W=256, C=4 为例，计算得 L=64，符合 GPU 采样对齐要求。

## Shader 实现建议

根据三维坐标 (x, y, z) 计算对应 2D 纹理 UV：

计算层索引：( \text{layerIndex} = \lfloor \frac{z}{4} \rfloor )

计算通道索引：( \text{channelIndex} = z \bmod 4 )

计算切片坐标：

[ \text{tileX} = \text{layerIndex} \bmod \frac{W}{L} ]

[ \text{tileY} = \left\lfloor \frac{\text{layerIndex}}{W/L} \right\rfloor ]

计算 UV 坐标：

[ U = \frac{\text{tileX} \times L + x}{W} ]

[ V = \frac{\text{tileY} \times L + y}{W} ]

可使用 step() 函数或位运算提取体素开关状态。
